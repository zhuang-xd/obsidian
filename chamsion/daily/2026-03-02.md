## debug

### 功能键无法停止录像

长按触发录像后设置了 press_cooldown_ticks 和 press_trigger_lock，短按检测需二者均为 0，导致短按在冷却期内被忽略。

因此在 recording_mode 下放宽press_cooldown_ticks和press_trigger_lock的值判断，允许短按立即停止录像。
```cpp
- if (!long_fired && press_ticks >= SHORT_PRESS_MIN_TICKS 
	&& !press_cooldown_ticks && !press_trigger_lock) {
	log_info("-------------------------TP SHORT----------------------------");
}

/* 录像模式下短按停止不受冷却期限制，可立即响应 */
+ int allow_short = recording_mode
+	? (!long_fired && press_ticks >= SHORT_PRESS_MIN_TICKS)
+	: (!long_fired && press_ticks >= SHORT_PRESS_MIN_TICKS && 
+		!press_cooldown_ticks && !press_trigger_lock);
+ if (allow_short) {
	log_info("-------------------------TP SHORT----------------------------");
}
```

语音开始录像后，没有更新 `recording_mode` 的状态

```cpp
case USER_UART_MSG19:
{
	if (rec_status == 0x01) {
		printf("---------------------Recording started-----------------------------\n");
		tp_set_recording_mode(1);  /* 同步录像状态，使功能键短按可停止录像 */
	} else if (rec_status == 0x00) {
		printf("----------------------Recording stopped-----------------------\n");
		tp_set_recording_mode(0);  /* 同步录像状态 */
	}
}
```
